
# PRD: Telegram‑бот для проведения взаимных собеседований

## 1. Общее описание

**Название продукта:** InterviewPartner Bot  
**Тип продукта:** Telegram‑бот + серверная часть на Java (Spring Boot / Telegram Java API).  
**Цель:** Помочь пользователям подготовиться к собеседованиям, проводя взаимные интервью друг с другом.  
AI‑агент в Cursor должен использовать этот PRD как основу для генерации и поддержки кода, моделей и архитектуры, а также использовать MCP серверы для ускорения разработки.

---

## 2. Цель и ценность для пользователя

- Пользователи практикуют собеседования в выбранном языке (например, английский, русский, Java‑технические вопросы).
- Можно выбрать день и время собеседования.
- Бот присылает напоминания, чтобы пользователь ничего не пропустил.
- Пользователи одновременно учатся как **кандидатам**, так и **интервьюерам**.

---

## 3. Целевая аудитория

- Программисты и соискатели, которые готовятся к техническим и поведенческим собеседованиям.
- Люди, которые хотят практиковать английский для собеседований.
- Люди, которые хотят учиться интервьюировать других (роль «ментора»).

---

## 4. Платформа и технологии

- Клиент: **Telegram** (бот‑интерфейс).
- Сервер: **Java** (рекомендуемо: Spring Boot + `TelegramBots` / `Telegram API Java`).
- Опционально: веб‑панель на **PrimeFaces** (для админки или просмотра расписаний).
- Хранение данных: **PostgreSQL / H2** (модели User, Interview, Schedule, Language, Role).

---

## 5. Основной функционал

Бот должен реализовывать следующие возможности:

| Функция | Краткое описание |
|--------|------------------|
| Регистрация | Пользователь впервые запускает бота — он регистрируется по Telegram‑ID и имени. |
| Просмотр собеседований | Список собеседований, где пользователь выступает в роли интервьюера или кандидата. |
| Создание собеседования | Пользователь создает собеседование: язык, формат (техническое / поведенческое), дата, время, длительность. |
| Поиск партнёра | Бот подбирает партнёра по языку и уровню (Junior/Middle/Senior). |
| Выбор языка | Пользователь выбирает язык собеседования (например: En‑Java, Ru‑SoftSkills). |
| Планирование расписания | Пользователь указывает, когда он может проводить или проходить собеседования (дни и время). |
| Напоминания | Бот отправляет сообщения‑напоминания перед собеседованием (например, за 15–30 минут). |
| История собеседований | Пользователь может просматривать прошедшие собеседования и их результаты (если будешь добавлять оценки). |

---

## 6. Пользовательские сценарии (User Stories)

AI‑агент должен понимать и поддерживать эти сценарии:

- «Как пользователь, я хочу создать собеседование, указав язык и время, чтобы я мог регулярно практиковаться».
- «Как пользователь, я хочу получать напоминания о собеседовании, чтобы не пропустить его».
- «Как пользователь, я хочу выбрать язык собеседования, чтобы тренироваться именно в нужном языке».
- «Как пользователь, я хочу найти партнёра для собеседования, чтобы не искать человека вручную».
- «Как пользователь, я хочу просматривать список своих будущих и прошедших собеседований».

---

## 7. Ожидаемые команды бота

Примеры Telegram‑команд:

- `/start` — начать работу с ботом, регистрация.
- `/create_interview` — начать создание собеседования (язык, формат, дата, время).
- `/find_partner` — искать партнёра для собеседования.
- `/schedule` — просмотр / изменение своего расписания.
- `/interviews` — список будущих и прошедших собеседований.
- `/help` — показать краткую справку по командам.

---

## 8. Основные модели данных (для Java‑части)

AI‑агент должен генерировать и поддерживать следующие сущности (Java‑классы):

- `User` — Telegram‑ID, имя, язык, уровень (Junior/Middle/Senior).
- `Interview` — связь между двумя пользователями, язык, формат, дата‑время, длительность, статус (запланировано / завершено).
- `Schedule` — блоки времени, когда пользователь может проводить или проходить собеседования.
- `InterviewHistory` — оценки, комментарии, результаты (по желанию).

Также можно добавить:
- `Notification` — записи о напоминаниях (когда и кому отправить).

---

## 9. Нефункциональные требования

| Требование | Описание |
|-----------|----------|
| Производительность | Ответ бота на команды — в пределах нескольких секунд. |
| Надёжность | Система не должна терять данные о собеседованиях и расписаниях. |
| Масштабируемость | Архитектура должна допускать рост числа пользователей и собеседований. |
| Безопасность | Хранение Telegram‑ID и персональных данных с учётом безопасности; токен бота — в переменных окружения. |
| Локализация | Поддержка как минимум русского и английского языков интерфейса. |

## Этап 1: Модели данных

### Задача 1.1: Модель User
**Поля:**
- id (Long, primary key)
- telegramId (Long, unique, not null)
- username (String)
- firstName (String)
- lastName (String)
- language (ENUM: ENGLISH, RUSSIAN)
- level (ENUM: JUNIOR, MIDDLE, SENIOR)
- createdAt (LocalDateTime)
- updatedAt (LocalDateTime)

**Тесты:**
- Создание пользователя с обязательными полями
- Валидация уникальности telegramId
- Обновление полей пользователя

### Задача 1.2: Модель Interview
**Поля:**
- id (Long, primary key)
- candidateId (Long, foreign key to User)
- interviewerId (Long, foreign key to User)
- language (ENUM: ENGLISH, RUSSIAN)
- format (ENUM: TECHNICAL, BEHAVIORAL)
- dateTime (LocalDateTime)
- duration (Integer, minutes)
- status (ENUM: SCHEDULED, COMPLETED, CANCELLED)
- createdAt (LocalDateTime)

**Тесты:**
- Создание интервью с валидными данными
- Проверка конфликта времени (нельзя создать два интервью одновременно)
- Обновление статуса интервью

### Задача 1.3: Модель Schedule
**Поля:**
- id (Long, primary key)
- userId (Long, foreign key to User)
- dayOfWeek (ENUM: MONDAY...SUNDAY)
- startTime (LocalTime)
- endTime (LocalTime)
- isAvailable (Boolean)

**Тесты:**
- Добавление доступного времени
- Проверка пересечения временных слотов
- Получение расписания пользователя

### Задача 1.4: Репозитории
- UserRepository (findByTelegramId, existsByTelegramId)
- InterviewRepository (findByUserId, findByDateTimeBetween, findConflictingInterviews)
- ScheduleRepository (findByUserId, findByDayOfWeek)

**Тесты:**
- Базовые CRUD операции
- Кастомные запросы
- Интеграция с БД

## Этап 2: Сервисный слой

### Задача 2.1: UserService
**Методы:**
- registerUser(telegramId, username) — регистрация нового пользователя
- getUserByTelegramId(telegramId) — получение пользователя
- updateUserLanguage(userId, language) — обновление языка
- updateUserLevel(userId, level) — обновление уровня

**Тесты:**
- Регистрация нового пользователя
- Получение существующего пользователя
- Обновление полей
- Обработка ошибок (пользователь не найден)

### Задача 2.2: InterviewService
**Методы:**
- createInterview(candidateId, interviewerId, language, format, dateTime, duration)
- findAvailablePartners(userId, language, dateTime)
- getUserInterviews(userId, status)
- cancelInterview(interviewId)
- completeInterview(interviewId)

**Тесты:**
- Создание интервью
- Поиск доступных партнеров
- Проверка конфликтов времени
- Отмена и завершение интервью

### Задача 2.3: ScheduleService
**Методы:**
- addAvailability(userId, dayOfWeek, startTime, endTime)
- removeAvailability(scheduleId)
- getUserSchedule(userId)
- isUserAvailable(userId, dateTime)

**Тесты:**
- Добавление доступности
- Удаление доступности
- Проверка доступности в конкретное время

## Этап 3: Telegram обработчики команд

### Задача 3.1: Базовый обработчик
- Создать класс CommandHandler
- Настроить маршрутизацию команд
- Добавить обработку неизвестных команд
- Написать тесты для маршрутизации

### Задача 3.2: Команда /start
**Функционал:**
- Приветственное сообщение
- Автоматическая регистрация пользователя
- Отображение главного меню с inline-кнопками

**Тесты:**
- Отправка /start новым пользователем
- Отправка /start существующим пользователем
- Корректность отображаемого меню

### Задача 3.3: Команда /create_interview
**Функционал:**
- Пошаговый диалог создания интервью
- Выбор языка (кнопки)
- Выбор формата (кнопки)
- Выбор даты и времени (inline календарь)
- Подтверждение создания

**Тесты:**
- Весь flow создания интервью
- Валидация введенных данных
- Обработка отмены на любом шаге

### Задача 3.4: Команда /find_partner
**Функционал:**
- Поиск доступных партнеров по языку и времени
- Отображение списка найденных партнеров
- Отправка запроса на интервью выбранному партнеру
- Обработка ответа (принять/отклонить)

**Тесты:**
- Поиск партнеров
- Отправка запроса
- Обработка ответа

### Задача 3.5: Команда /interviews
**Функционал:**
- Отображение списка предстоящих интервью
- Отображение истории прошедших интервью
- Детальная информация по каждому интервью
- Кнопка для отмены предстоящего интервью

**Тесты:**
- Получение списка интервью
- Пагинация (если интервью много)
- Отмена интервью

### Задача 3.6: Команда /schedule
**Функционал:**
- Просмотр текущего расписания
- Добавление новых временных слотов
- Удаление существующих слотов
- Визуальное отображение недельного расписания

**Тесты:**
- Просмотр расписания
- Добавление слота
- Удаление слота

- Реализовать проверку предстоящих интервью каждую минуту

**Тесты:**

### Задача 4.2: Отправка напоминаний
- Напоминание за 24 часа до интервью
- Напоминание за 1 час до интервью
- Напоминание за 15 минут до интервью
- Отметка о отправленных напоминаниях (чтобы не дублировать)

**Тесты:**
- Отправка напоминаний в правильное время
- Нет дублирования напоминаний
- Обработка ошибок отправки

4. Реализовать функционал
